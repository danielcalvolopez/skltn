All four phases (Skeleton Engine, MCP Server, Observability Layer, Web Dashboard) must have their design specs and implementation plans written and approved before any implementation begins.

**Why:** The user wants a complete architectural picture across all phases to catch cross-phase design issues early and avoid rework.

**How to apply:** After completing each phase's spec + plan, move to the next phase's brainstorming — do not start coding. Only begin implementation after all four phases are planned.
